# 领域上下文

本文档是项目的领域词汇表，只记录业务语言和协作概念，不记录实现细节、计划草稿或架构说明。

## 术语表

| 术语 | 含义 | 避免 / 备注 |
|---|---|---|
| OpenSpec | 基于 OpenAPI 3.1 的 API 契约层。 | 讨论前后端 API 契约时使用。 |
| Agent | 执行特定工作流步骤的 AI 协作者。 | 不要与人工审查者混用。 |
| PRD | 产品需求文档，用于记录用户问题、解决方案、用户故事、关键决策和测试 seam。 | 除非作为既有参考，否则不要写入具体实现文件路径。 |
| 垂直切片（Vertical Slice） | 贯穿所有受影响层、可独立验证的窄功能路径。 | 优先使用垂直切片，避免只按层拆分的横向任务。 |
| ADR | 架构决策记录，用于沉淀难以回滚、非显而易见且存在真实取舍的技术决策。 | 常规实现选择不要写 ADR。 |
| Comet Change | 承载一次 OpenSpec / Comet 生命周期推进的变更单元，包含 proposal、design、tasks、spec 和状态文件。 | 不要把具体产品样例 change 固化在模板中。 |
| Fresh Verification | 完成前重新执行的验证证据，包括测试命令、契约校验、关键路径检查或人工审查结论。 | 不等同于“之前跑过”或实现者自述。 |
| 流程模板资产 | 可被不同项目复用的文档模板、技能说明、校验脚本、流程规范和目录占位文件。 | 不包含某个具体产品的 PRD、设计、OpenAPI、Issue 或 OpenSpec change。 |
| Harness 工程 | 串联研发生命周期阶段、门禁、产物、Agent/Skill 编排和追踪证据的工程化治理载体。 | 不等同于单个 CI 流水线、单个 Comet Change 或零散提示词集合。 |
| 研发管理仓库 | 承载 PRD、OpenAPI、OpenSpec / Comet、架构、垂直切片、验证、发布和复盘等研发管理资产的仓库。 | 不等同于前端 / 后端代码 monorepo。 |
| 实现仓库 | 承载前端、后端或其他运行时代码及其 Git、CI、MR / PR、测试命令和发布流水线的仓库。 | 不要把实现仓库的源码所有权混入研发管理仓库。 |

只有在计划、分诊、调试或架构讨论中明确沉淀出稳定语言时，才新增术语。
